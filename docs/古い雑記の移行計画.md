# 古い雑記の移行計画

## 概要

`../DLB/`ディレクトリにある2000年〜2014年の雑記HTMLファイルを新しいAstroベースのブログシステムに移行する計画です。

## 移行対象ファイル

### 雑記ファイル一覧
- `zakki.htm` - 2014年（現在進行形）
- `zakki13.htm` - 2013年
- `zakki12.htm` - 2012年
- `zakki11.htm` - 2011年
- `zakki10.htm` - 2010年
- `zakki09.htm` - 2009年
- `zakki08.htm` - 2008年
- `zakki07.htm` - 2007年
- `zakki06.htm` - 2006年
- `zakki05.htm` - 2005年
- `zakki04.htm` - 2004年
- `zakki03.htm` - 2003年
- `zakki02.htm` - 2002年
- `zakki01.htm` - 2001年
- `zakki00.htm` - 2000年

**合計**: 15ファイル（約15年分の記事）

## 現在の記事構造分析

### HTMLファイルの構造
```html
<div id="main">
  <h2>YYYY/MM/DD</h2>
  <p>記事内容...</p>
  
  <h2>YYYY/MM/DD</h2>
  <p>記事内容...</p>
</div>
```

### 記事の特徴
- **日付形式**: `YYYY/MM/DD`（例：2014/06/24）
- **記事区切り**: `<h2>`タグで日付を表示
- **内容**: `<p>`タグ内にHTML形式で記述
- **リンク**: 外部サイトへのリンクが多数含まれる
- **文字装飾**: `<b>`, `<font>`, `<br>`タグを使用
- **年代順**: 各ファイル内では新しい記事が上部に配置

#### 1.2 マークダウン変換機能
- HTMLタグをマークダウン記法に変換
- `<br>` → 改行
- `<b>` → `**太字**`
- `<a href="">` → `[テキスト](URL)`
- `<font color="">` → 適切なマークダウン記法に変換

#### 1.3 メタデータ生成

### Phase 2: 自動タグ付けシステム

#### 2.1 コンテンツベースタグ抽出
記事内容から自動的にタグを生成：

**ゲーム関連**
- DDR, ビートマニア → `["音ゲー", "ゲーム"]`
- オーフェン → `["アニメ", "小説", "オーフェン"]`
- モンコレ → `["カードゲーム", "モンスターコレクション"]`

**技術関連**
- HTML, CSS, JavaScript → `["Web開発", "プログラミング"]`
- PHP → `["サーバーサイド", "プログラミング"]`

**生活関連**
- 学校, テスト → `["学生生活", "日常"]`
- 就職活動 → `["就職", "仕事"]`

#### 2.2 年代別タグ
- 2000年代前半 → `["2000年代", "学生時代"]`
- 2000年代後半 → `["2000年代", "社会人"]`
- 2010年代 → `["2010年代"]`

### Phase 3: データクリーニング

#### 3.1 リンク修正

- **無効リンク処理**: 取り消し線表示とツールチップでの詳細情報
- **リンク管理**: 定期的なチェック機能と修正履歴の管理

- @astrojs/imageによる自動最適化とWebP変換
- 手動での品質チェックが可能な規模

#### 3.3 文字エンコーディング
- 文字化け修正
- 特殊文字の正規化

## 技術的考慮事項

### 移行ツールの技術スタック
- **言語**: TypeScript
- **HTMLパーサー**: cheerio
- **マークダウン変換**: turndown
- **ファイル操作**: Node.js fs module

### 移行時の課題と対策

#### 1. 文字エンコーディング問題
- **問題**: 古いHTMLファイルの文字化け
- **対策**: 複数エンコーディングでの読み込み試行

#### 2. HTML構造の不整合
- **問題**: 一部ファイルでHTML構造が異なる可能性
- **対策**: 柔軟なパーサー実装、手動確認フロー

#### 3. 大量データの処理
- **問題**: 15年分の記事データ処理
- **対策**: バッチ処理、進捗表示、エラーハンドリング

#### 4. 外部リンクの管理
- **問題**: 古いリンクの多くが無効
- **対策**: リンクチェッカー実装、アーカイブサイト活用

#### 5. リンク処理の詳細実装
