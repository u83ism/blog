# ブログシステム仕様書

## プロジェクト概要

フロントエンドのみで完結する静的サイト生成（SSG）ベースのブログシステム

## 技術要件

### 必須技術
- **言語**: TypeScript
- **フレームワーク**: Gatsby（React ベース）
- **スタイリング**: CSS Modules または Styled Components
- **ビルドツール**: Gatsby CLI
- **デプロイ**: Netlify / Vercel / GitHub Pages

### 推奨技術
- **マークダウン処理**: gatsby-transformer-remark
- **画像最適化**: gatsby-plugin-image
- **SEO**: gatsby-plugin-react-helmet
- **PWA対応**: gatsby-plugin-manifest, gatsby-plugin-offline

## 機能要件

### コア機能

#### 1. 記事管理
- マークダウンファイルでの記事作成
- フロントマター（メタデータ）による記事情報管理
  - タイトル
  - 作成日時
  - 更新日時
  - タグ
  - 概要
  - 公開状態

#### 2. フィルタリング機能
- **タグフィルタ**
  - 複数タグの選択可能
  - AND/OR検索の切り替え
  - タグクラウド表示
- **月別フィルタ**
  - 年月別のアーカイブ表示
  - カレンダー形式での選択
  - 記事数の表示

#### 3. 検索機能
- タイトル・本文での全文検索
- 検索結果のハイライト表示

### UI/UX機能

#### 1. レスポンシブデザイン
- モバイルファースト設計
- タブレット・デスクトップ対応

#### 2. ナビゲーション
- ヘッダーナビゲーション
- サイドバー（タグ一覧、月別アーカイブ）
- パンくずリスト
- ページネーション

#### 3. 記事表示
- 記事一覧ページ
- 記事詳細ページ
- 関連記事の表示
- 前後記事へのナビゲーション

### パフォーマンス要件

#### 1. 読み込み速度
- 初回読み込み: 3秒以内
- ページ遷移: 1秒以内
- 画像の遅延読み込み

#### 2. SEO対応
- メタタグの自動生成
- サイトマップの自動生成
- 構造化データの実装

```
blog/
├── src/
│   ├── components/          # 再利用可能なコンポーネント
│   │   ├── layout/         # レイアウト関連

│   └── styles/             # グローバルスタイル
├── content/
│   └── blog/               # マークダウン記事
├── static/                 # 静的ファイル
├── docs/                   # プロジェクト資料
└── gatsby-*.js             # Gatsby設定ファイル
```

## データ構造

### 記事のフロントマター
```yaml
---
title: "記事タイトル"
date: "2024-01-01"
updated: "2024-01-02"
tags: ["JavaScript", "React", "TypeScript"]
description: "記事の概要"
published: true
slug: "article-slug"
---
```

### GraphQLクエリ例
```graphql
query BlogPosts {
  allMarkdownRemark(
    sort: { fields: [frontmatter___date], order: DESC }
    filter: { frontmatter: { published: { eq: true } } }
  ) {
    nodes {
      id
      frontmatter {
        title
        date
        tags
        description
        slug
      }
      excerpt
      timeToRead
    }
  }
}
```

## 開発フロー

### 1. 環境構築
```bash
npm install -g gatsby-cli
gatsby new blog-system
cd blog-system
npm install
```

### 2. 開発サーバー起動
```bash
gatsby develop
```

### 3. ビルド・デプロイ
```bash
gatsby build
gatsby serve  # ローカル確認
```

## コーディング規約

### TypeScript
- 厳密な型定義を心がける
- `readonly` を積極的に使用
- `interface` や `class` は極力避け、型エイリアスを使用

### React/Gatsby
- 関数型コンポーネントを使用
- カスタムフックで状態管理ロジックを分離
- TSXのreturn部分にロジックを埋め込まない

### ファイル命名
- ケバブケースで統一
- コンポーネントファイルは `.tsx`
- ユーティリティファイルは `.ts`

### ファイルサイズ
- 1ファイル100-150行を目安
- 超過する場合は機能単位で分割
