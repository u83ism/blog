```html
<div id="main">
  <h2>YYYY/MM/DD</h2>
  <p>本文……<br>

```

## 変換方針

### ファイル命名・配置

```
src/content/posts/YYYY/YYYY-MM-DD.md
```

同一日付が複数ある場合（稀）：`YYYY-MM-DD-2.md` のように連番サフィックス。

### HTMLタグ変換ルール

| 元 | 変換後 | 備考 |
|---|---|---|
| `<h2>YYYY/MM/DD</h2>` | frontmatter `date:` + ファイル名 | |
| `<b>text</b>` | `**text**` | |
| `<a href="url">text</a>` | `[text](url)` | リンク切れはそのまま残す |
| `<br>` | 削除（前後テキストを繋げる） | |
| `<br>` + `<br>`（連続） | 段落区切り（空行）に変換 | `<br>\n　<br>` パターンも同様 |
| `<font color="...">text</font>` | タグ除去、テキストのみ残す | 色情報は捨てる |
| `<font size="..." color="..."><b>text</b></font>` | `**text**` | |
| `（後日追記：...）` | `> 後日追記：...` （blockquote） | 構造として保持 |
| 全角スペース（`　`） | そのまま保持 | 段落頭の字下げ |
| `<ul>` / `<li>` | `- item` | |

### 変換スクリプト

TypeScriptで実装（Node.js）。  
`DLB/scripts/convert-zakki.ts` として作成し、実行後の出力を `blog/src/content/posts/` に配置する。

主な処理フロー：
1. `legacy/zakki*.htm` を読み込み
2. `#main` 内の `<h2>` を起点にエントリを分割
3. 各エントリの `<p>` 内容をMarkdownに変換
4. frontmatterを付与して `.md` ファイルとして書き出し

使用ライブラリ候補：
- `node-html-parser` または `cheerio`（HTML解析）
- 変換処理は軽量なので外部ライブラリ最小限で実装

## タグ付け方針

自動タグ付けは精度が低いため、変換後に手動で付与する。

### 事前設計する主要タグ（候補）

| タグ | 対象内容の例 |
|---|---|
| `ゲーム` | RPG・アクション等のゲームプレイ記録 |
| `音ゲー` | DDR・ビートマニア等 |
| `プロレス` | 観戦・試合感想 |
| `アニメ` | アニメ・マンガ感想 |
| `日常` | 学校・仕事・生活 |
| `サイト運営` | HP更新・技術 |
| `Web技術` | CSS・JavaScript・ブラウザ挙動等 |

タグを最終決定してから一括付与作業を行う。
