## 変換方針

### ファイル命名・配置

```
src/content/posts/YYYY/YYYY-MM-DD.md
```

同一日付が複数ある場合（稀）：`YYYY-MM-DD-2.md` のように連番サフィックス。

### HTMLタグ変換ルール

| 元 | 変換後 | 備考 |
|---|---|---|
| `<h2>YYYY/MM/DD</h2>` | frontmatter `date:` + ファイル名 | |
| `<b>text</b>` | `**text**` | |
| `<a href="url">text</a>` | `[text](url)` | リンク切れはそのまま残す |
| `<br>` | 削除（前後テキストを繋げる） | |
| `<br>` + `<br>`（連続） | 段落区切り（空行）に変換 | `<br>\n　<br>` パターンも同様 |
| `<font color="...">text</font>` | タグ除去、テキストのみ残す | 色情報は捨てる |
| `<font size="..." color="..."><b>text</b></font>` | `**text**` | |
| `（後日追記：...）` | `> 後日追記：...` （blockquote） | 構造として保持 |
| 全角スペース（`　`） | そのまま保持 | 段落頭の字下げ |
| `<ul>` / `<li>` | `- item` | |

### 変換スクリプト

TypeScriptで実装（Node.js）。  
`DLB/scripts/convert-zakki.ts` として作成し、実行後の出力を `blog/src/content/posts/` に配置する。

主な処理フロー：
1. `legacy/zakki*.htm` を読み込み
2. `#main` 内の `<h2>` を起点にエントリを分割
3. 各エントリの `<p>` 内容をMarkdownに変換
4. frontmatterを付与して `.md` ファイルとして書き出し

使用ライブラリ候補：
- `node-html-parser` または `cheerio`（HTML解析）
- 変換処理は軽量なので外部ライブラリ最小限で実装

### タグの管理方法

タグ名は **`src/lib/tags.ts` の `TAGS` 配列のみで管理** する。
ここを変更すれば、スキーマ検証・スクリプト・型定義がすべて自動で追従する。

#### タグを追加する手順

1. **`src/lib/tags.ts`** の `TAGS` 配列に新しいタグ名を追記する

   ```typescript
   export const TAGS = [
     // ... 既存タグ ...
     '新タグ名',  // ← 追加
   ] as const;
   ```

2. **`scripts/assign-tags.ts`** の `TAG_RULES` に判定キーワードを追加する

   ```typescript
   {
     tag: '新タグ名',
     keywords: ['キーワードA', 'キーワードB'],
   },
   ```

3. スクリプトを実行して既存記事に一括付与する

   ```bash
   # 変更内容を確認（ファイルは変更しない）
   npx tsx scripts/assign-tags.ts --dry-run

   # 実際に書き込む
   npx tsx scripts/assign-tags.ts
   ```

#### タグを手動で付与・修正する

記事の frontmatter を直接編集する。タグ名は `TAGS` に定義されたもの以外は
ビルド時にエラーになる（タイポ防止）。

```yaml
---
date: 2000-04-08
tags: [ゲーム, 音ゲー, 日常]
---
```

#### タグを削除する手順

1. `src/lib/tags.ts` から該当タグを削除する
2. そのタグを持つ記事の frontmatter を一括修正する（削除するか別タグに変更）
3. `scripts/assign-tags.ts` の `TAG_RULES` からも削除する

#### 未タグ記事を確認する

```bash
# タグなし記事のパスを一覧表示
grep -rl "tags: \[\]" src/content/posts/
```
