# 開発ワークフロー

## 手順

1. **事前準備**
   - `README.md` および `docs/` 配下を読み、仕様と制約条件を整理する。

2. **実装**
   - 実装計画を作成し、コーディング規約に従ってコードを生成する。

3. **テスト**
   - `src/lib/` 配下にビジネスロジックが含まれる場合のみ、テストを生成・実行する。
   - Astroページ・コンポーネントのみの変更であればスキップしてよい。

4. **進捗ドキュメント更新**
   - `docs/` 内の進捗に関する記述（実装済み機能・残タスクなど）を実装内容に合わせて更新する。
   - 仕様の整合性チェックは行わない。

## 使用例
- 「新しいページを追加して。」
- 「src/lib/ に変換ロジックを追加して。」
